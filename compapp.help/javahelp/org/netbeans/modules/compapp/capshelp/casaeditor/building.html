Building a Composite Application

See Also

When you build a composite application, you create a service assembly. The service assembly contains service units representing each of the service engines or binding components that make up the composite application. After you create the service assembly, you can deploy the service assembly to the application server.

You can build a composite application from either the composite application project in the NetBeans Project window, or from within the CASA Editor. You typically build the composite application from within the CASA Editor for the following reasons:

  • You have a new composite application that you have not previously built.
    The CASA Editor cannot display the relative layout of the components unless you build the composite application.
  • You have modified the configuration of a composite application.
    You can then deploy the composite application with the modifications you have made.

To build a composite application in the CASA Editor:

  1. From the Editor Toolbar, select the Build Project button.
    If this is the first time the composite application has been built, the CASA Editor arranges the components and their connections in the Design Area of the editor.
