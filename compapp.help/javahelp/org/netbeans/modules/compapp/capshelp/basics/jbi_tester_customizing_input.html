Customizing Test Input

See Also

To customize test input:

  1. In the project tree under your composite application's Test node > Test Case, right-click Input and click Edit.
  2. Modify its contents as needed. For example, wherever you see <value>?string?</value> click within ?string? and replace it with a string of any length. However, within such strings, do not include the characters < (less-than sign) or & (ampersand) unless you use them with XML semantics.
  3. When you are satisfied, click Save.
  4. Right-click Output and click Edit to examine its contents:
    • It is empty. This is a special state that triggers a special operation when the test is run.
    • Each time the test is run, the current output is compared to the contents of Output.xml; whatever differences are detected will be stored in the Actual_yymmddhhmmss.xml file under the test case folder. However, in the special case where Output.xml starts null, then the output is written to the Output.xml file.
    • In each run after the first, assuming Output.xml is no longer null, its contents are preserved. In other words, a previous output is never overwritten by new results.
