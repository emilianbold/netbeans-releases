BPEL Debugger: Threads Window

See Also

The BPEL process often has more than one thread . Multiple threads appear if the process includes either Flow or On Event handler.

The Threads window lists all threads for the current process instance in the current debugging session. You open the Threads window by choosing Window > Debugging > Threads.

The information given for each thread is the following:

  • Name. Thread name
  • State. Shows the current state of the thread. It can be either ACTIVE or COMPLETED.

If the current process instance is suspended, the current thread is marked with bold.

When you select a different current session, the Threads window is updated to show the threads for that session.
