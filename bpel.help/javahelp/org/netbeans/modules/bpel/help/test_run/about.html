Deploying and Performing Test Runs of BPEL Processes

See Also

This topic describes the tasks you have to perform to deploy and test-run your business process.


Build
BPEL Module project
  • (Optional) Build your BPEL Module project.


Create CA project

  • Create a Composite Application project if you have not already done so. For sample processes, the IDE creates this type of projects automatically.

Add BPEL Module project
  • Add the BPEL Module as a JBI Module to the Composite Application project.

Build CA project
  • (Optional) Build the Composite Application project.

Start application server
  • Make sure that the Application Server is started.

Deploy CA project
  • Deploy the Composite Application project.

Create and run test cases
  1. Create test cases (for sample processes, test cases are automatically created; for new projects, you need to create at least one test case).
  2. Run one or several test cases.
  • You can also debug your BPEL process after it is deployed to the server. See About Debugging BPEL Processes for more information on how to debug BPEL applications.
