Enabling the UserNameToken Security Mechanism for Web Service Providers

See Also

You use the Security tab of the Web Service Attributes Editor to enable and configure the UserNameToken security mechanism for web service providers.

To enable the Liberty security mechanism for a web service provider:

  1. In the Projects window, expand the project node for the project where your web service provider is defined.
  2. Expand the Web Services node, right-click the web-service node, and choose Edit Web Service Attributes from the pop-up menu.
    The Web Service Attributes Editor opens with the Security tab open.
  3. Select the Enable Message Level Security checkbox.
  4. Under Security Mechanism, from the Request drop-down list, select UserNameToken.
  5. Click OK.
  • You must configure the list of username and password combinations of the web service clients that access this provider. For instructions, see Configuring Security Mechanisms for Sun Java System Access Manager Instances.
