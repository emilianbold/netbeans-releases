SJS Application Server Tasks: Quick Reference

See Also

This topic describes common tasks you can perform with applications and resources on the SJS Application Server. For more detailed information, click the links in the right column.

To perform this task Follow these steps
Register the SJS Application Server with the IDE.
  1. Choose Tools  Server Manager.
  2. Click Add Server to start the Add Server wizard.
  3. Choose the server type, give the server instance a name and click Next.
  4. Define the server instance properties and click Finish.

For more information, see Registering the SJS Application Server.
Start or stop the server.
  1. In the Runtime window, right-click the SJS Application Server node and choose Start / Stop Server.
  2. Click Start Server or Stop Server in the dialog box.
Create a JDBC connection pool.
  1. Choose File  New (Ctrl-N) to open the New File wizard.
  2. Choose the EJB module from the project combo box.
  3. Select JDBC Connection Pool from the Sun Resources category and click Next.
  4. Choose a connection and enter details for the connection pool.
  5. Click Finish.

For more information, see Setting up a Connection Pool on the SJS Application Server.
Add a JDBC resource.
  1. Choose File  New (Ctrl-N) to open the New File wizard.
  2. Choose the EJB module from the combo box.
  3. Select JDBC Resource from the Sun Resources category and click Next.
  4. Select to use an existing connection pool or create a new one.
  5. Enter a JNDI name and any other details.
  6. Click Finish.

For more information, see Setting up a JDBC Resource on the SJS Application Server.
Add a JMS resource.
  1. Choose File  New (Ctrl-N) to open the New File wizard..
  2. Choose the EJB module from the combo box.
  3. Select JMS Resource from the Sun Resources category and click Next.
  4. Enter a JNDI name for the resource and any other details.
  5. Choose a resource type and click Next.
  6. Enter any resource properties.
  7. Click Finish.

For more information, see Setting up a JMS Resource on the SJS Application Server.
Add a JavaMail resource.
  1. Choose File  New (Ctrl-N) to open the New File wizard..
  2. Choose the EJB module from the combo box.
  3. Select JavaMail Resource from the Sun Resources category and click Next.
  4. Enter a JNDI name and the server and user details.
  5. Click Finish.

For more information, see Setting up a JavaMail Resource on the SJS Application Server.
Add a persistence resource.
  1. Choose File  New (Ctrl-N) to open the New File wizard..
  2. Choose the EJB module from the combo box.
  3. Select Persistence Resource from the Sun Resources category and click Next.
  4. Select to use an existing JDBC resource or create a new one.
  5. Enter a JNDI name and any other details.
  6. Click Finish.

For more information, see Setting up a Persistence Manager Resource on the SJS Application Server.
Undeploy an application.
  1. Expand the Sun Java System Application Server instance and the Applications node in the Runtime window.
  2. Expand the node which contains the type of application to undeploy.
  3. Right-click the application and choose Undeploy from the contextual menu.

For more information on undeploying an application and other contextual menu items, see Removing an Application from a Server.
