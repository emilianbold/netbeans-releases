Working With Dependencies and Extension Points

The UML Modeling feature enables you to set up certain relationships within a Use Case diagram:

  • Create Elements and Associations
  • Add an Include Dependency
  • Add an Extend Dependency
  • Add Extension Points

The following exercise illustrates each kind of relationship. Start with an empty Use Case diagram.

Creating Elements and Associations

  1. Create two Actors named Customer and Bank.
  2. Create three Use Cases named Withdraw Money, Transfer Money, and Deposit Money.
  3. Associate Customer with all three Use Cases by dragging an Association from Customer to each Use Case.
    In the Modeling Palette window, click the Association icon and drag from one element to the other.
  4. In the same way, associate Bank with all three Use Cases.

Adding an Include Dependency

  1. Create another Use Case named Cash.
  2. In the Modeling Palette, click the Include icon .
  3. In the diagram, drag the Include relationship from Cash to Withdraw Money.

    Notice that the relationship is stereotyped as <<include>>.
    • You can move the stereotype around the diagram without losing the stereotype's integrity. You can also hide stereotype labels.
  4. In the diagram, right-click the relationship and choose Labels > Stereotype.

Adding an Extend Dependency

  1. Create another Use Case named Money Market.
  2. In the Modeling Palette, click the Extend icon .
  3. In the diagram, drag an Extend relationship from Money Market to Deposit Money.

    Notice that the relationship is stereotyped as <<extend>>.
  4. In the diagram, right-click the relationship and choose Labels > Stereotype.

Adding Extension Points

  1. In the diagram, select Deposit Money.
  2. In the Properties window, select the Extension Points row.
  3. In the Extension Points dialog box, click Add and type Name.
    The IDE names the extension point.
  4. To add more documentation to the extension point, in the Properties window, choose the Documentation field to open the Advanced Documentation Editor.
    The IDE creates a values compartment inside the Use Case element that displays the extension points.
  5. To see the include or exclude relationship with its corresponding Use Case, navigate to the Use Case in the Projects window and expand the nodes underneath it.
