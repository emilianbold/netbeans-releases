Integrating Support for the DOORS Tool

Support for Telelogic's DOORS requirements tool (versions 7.0 and 6.0) is available for integration in the IDE. This support lets you drag requirements from the DOORS tool onto UML Modeling Elements, for traceability. The IDE populates the Requirements section in the UML Design Center with a list of DOORS requirements. Note that this tool is only available for the Windows platform.

To integrate support for the DOORS tool:

  1. On a Windows platform, locate the configDoors.vbs script file.
    If you performed a local install of the UML module, the script file can be found in <NetBeans_IDE_User_Dir>\5.5\modules\DoorsIntegrationFiles\configDoors.vbs.
    For example, C:\Documents and Settings\Administrator\.netbeans\5.5\modules\DoorsIntegrationFiles\configDoors.vbs

    If you perfomed a global install of the UML module, the script file is located in <Netbeans_IDE_Install_Dir>\nb5.5\modules\DoorsIntegrationFiles\configDoors.vbs
  2. Open a command line window and go to the directory where the configDoors.vbs script file is located.
  3. Execute the configDoors.vbs script as follows: 
    C:\windows\system32\cscript.exe configDoors.vbs
    A successful run of the script completes the following steps:
    • copy the DOORS integration files into the Telelogic DOORS lib directory
    • create a system registry key and value for the DOORS tool
    • add the path to the DOORS integration .dll file in the system PATH environment variable.
