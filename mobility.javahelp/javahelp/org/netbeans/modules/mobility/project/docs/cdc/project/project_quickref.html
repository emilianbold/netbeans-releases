CDC Tasks: Quick Reference

This topic describes common tasks you can perform with the NetBeans Mobility Pack for CDC.

To perform this task Follow these steps
Create a project
  1. Choose File > New Project.
    This opens the New Project wizard.
  2. Under Categories, Choose CDC
  3. Under Projects, choose the CDC template that best fits the needs of your project.
Configure a project
  1. Right click on a project and choose Platforms.
  2. Choose a platform from the Java Platform drop-down menu.
    Note: If you change the platform to one using a different profile, you will lose your GUI builder palette settings.
  3. Choose a device from the Device drop-down menu.
  4. Choose a profile from the Profile drop-down menu.
Run a project
  1. Choose Run > Run Main Project (F6) or right click on any project in the Projects window and choose Run Project.
Debug a project
  1. To debug the main project, choose Run > Debug Main Project (F5).
  2. To debug an individual project, right-click the project and choose Debug Project.
Run Samples
  1. Choose File > New Project.
    This opens the New Project wizard.
  2. Under Categories, Choose Samples > CDC Then select the folder that contains the sample project you wish to run .
  3. In the Projects window choose the sample project you want to run.
  4. Select the project configuration properties as desired.
  5. When the project is loaded, right-click and choose Run Project.
Create CDC application interfaces using the GUI builder

The GUI Builder enables you to create menus and other UI elements by enabling you to place components where you want them and providing visual feedback guidelines.

The GUI Builder opens when you create a new project for the Personal and AGUI profiles.

To open the GUI builder in an open project:

  1. Right-click on a project and choose New File/Folder.
  2. Under Categories, choose CDC. Under File Types, choose AGUI Xlet Form or Personal Profile Xlet Form.

    Note: The GUI Builder is not available for projects using the Ricoh (Foundation Profile) Platform Emulator.

 
Add an emulator platform
  1. Download and install the Emulator you want to add.
  2. Choose Tools > Java Platform Manager Click the Add Platform button
  3. In the Add Platform dialog box select Java Micro Edition CDC Platform Emulator
  4. Select the directory where the platform you want to add was installed.
  5. Add the emulator.
Add or remove files from the JAR
  1. Right-click the project you want to edit and choose Properties.
  2. Select Packaging under Categories.
    In the Project Properties dialog box you can set the following properties:
    • JAR File. The name of the distribution jar file.
    • Exclude from JAR file. Specifies which files to exclude from the distribution jar.
Manage resources in a project.

CDC application resources can be located outside of the project JAR, but must be physically located in the <project>/resources folder.

To add resources to a CDC project:

  • Drag and drop resources in the Projects or Files views.
  • Cut and paste into the Resources folder anywhere in the logical view.
Add custom libraries
  • Select Tools > Library Manager.
  • In the Library Manager dialog box click the New Library button.
  • In the New Library window, type a name for the library.
  • For Library Type select CDC Class libraries in the drop-down menu and click OK. The custom library you created will be in the CDC class Libraries window.
  • Click on the custom library then click the Classpath tab.
  • Click the Add JAR/Folder button and choose the directory your new library is located in.
