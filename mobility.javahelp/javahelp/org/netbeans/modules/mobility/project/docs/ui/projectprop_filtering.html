MIDP Project Properties: Sources Filtering

See Also

Use the Sources Filtering properties page to determine the files and packages that are included in a JAR file for a specific .

You open the Sources Filtering properties page by choosing File > <active project name> Properties. In the Project Properties dialog, select Build > Sources Filtering.

  • If you are setting the properties for a project configuration other than the default configuration (DefaultConfiguration), you must uncheck the Use Values from "DefaultConfiguration" checkbox at the top of the page to customize these options.

The Filtering page lets you set the following options:

  • Exclude Files That Ant Excludes By Default. If checked, files that are defined by Ant to be excluded by default (version control files and folders) are excluded from the JAR file.
  • Exclude Test Sources. If checked, tests and test packages are excluded from the JAR file.

To see the list of files excluded by these two options, hold the cursor over the option. The tooltip lists the files to be excluded.

The Sources Used in Build window provides a file tree where you can identify files and packages to be included or excluded from the JAR file. Top level nodes are top level sources and packages. All files and packages are selected by default.

To include or exclude an item from the build:

  • All items are included by default. Expand the nodes and uncheck the boxes next to the files or packages you want to exclude.
