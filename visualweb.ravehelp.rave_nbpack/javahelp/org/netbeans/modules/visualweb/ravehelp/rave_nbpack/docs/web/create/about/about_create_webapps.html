About Creating Web Applications

See Also

Web applications have a defined folder structure. A web application is usually packaged and deployed as a web archive (WAR) file, although a web application does not always have to be packaged in order to be deployed.

A web application must contain a deployment descriptor file (WEB-INF/web.xml). A web application can contain one or more web components.

A visual web application is a project that uses the Visual Web JavaServer Faces framework. When you create a Visual Web project, the IDE does the following:

  • Displays the new project in the If the Set as Main Project checkbox was selected in the New Project wizard, the new project becomes the main project. If necessary, you can change the main project.
  • Opens a page for you to edit in the Visual Designer.
  • Puts all the files in the $HOME/NetBeansProjects/project-name directory on a UNIX® or Mac OS system or in the \Documents and Settings\username\My Documents\NetBeansProjects\project-name folder on a Microsoft Windows system.

After you create the project, you can view the files in the Projects window or the Files window.

  • You can add Visual Web JavaServer Faces pages to a web application project that does not initially include the Visual Web JavaServer Faces framework. To do so, right-click the project name node in the Projects window and choose New > Visual Web JSF Page.
