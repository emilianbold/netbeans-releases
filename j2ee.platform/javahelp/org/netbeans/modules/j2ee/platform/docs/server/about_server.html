About Servers

See Also

A server is a computer that serves up applications. Any computer can be turned into a server if you install server software and connect the computer to the Internet. The following servers are supported by the IDE:

  • SJS Application Server

    Use this server and its libraries for the following activities:

    • Deploying enterprise applications and enterprise beans.
    • Deploying web services and web service clients that use JSR-109 or JSR-101 stubs.
    • Deploying web applications, JSP pages, and servlets.
    • Libraries included with the SJS Application Server can be used to compile web services and clients, verify that your application implements the J2EE 1.4 specification correctly, create a new web application with JavaServer Faces (JSF) support, or add JSF support to an existing web application
  • Tomcat Web Server

    Use this server and its libraries for the following activities:

    • Deploying web applications, JSP pages, and servlets.
    • Deploying web services and web service clients that use JSR-101 stubs.
        The Tomcat Web Server is not a J2EE 1.4 container. Therefore, you cannot deploy a web service client that uses JSR-109 stubs to this server. When you create the web service client, make sure that you select JAXRPC Static Stub in the Web service client type drop-down.

Servers in the IDE

Before you can deploy an enterprise application, web application, JSP, servlet, or EJB module, the server to which you are going to deploy needs to be registered with the IDE. If you download the NetBeans IDE 4.1 Installer, the Tomcat Web Server is registered with the IDE. If you download the NetBeans IDE 4.1 + AS 8.1 PE Bundle Installer, both the Tomcat Web Server and the SJS Application Server is registered with the IDE.

  Only once you have a registered SJS Application Server can you begin developing enterprise applications.

When you register a server with the IDE, you make its libraries available for production, or deployment, or both. If you deploy your applications to a remote instance of the SJS Application Server, its libraries are available at runtime. However, during development, you might need a local instance of this server. You can have multiple instances of the SJS Application Server registered with the IDE. Once you have registered your local instance, you can register remote instances, but you always have to specify the local folder that contains the local installation of the SJS Application Server.

When a server is registered with the IDE, you can see its node in the Runtime window under the the Servers node. When you create a project in the New Project wizard, you select the server to which you want to deploy your application. After you create the application, you can change the server by right-clicking the project, choosing Properties, clicking Run, and selecting a different server.
