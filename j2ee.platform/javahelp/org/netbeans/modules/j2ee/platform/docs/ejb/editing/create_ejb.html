Creating an Enterprise Bean

See Also

  1. In the Projects window, select any existing EJB module project.
  2. Choose File > New from the main window.
  3. From the Enterprise category, select one of the following EJB templates:
    • Session Bean
    • Message-Driven Bean
    • Entity Bean (J2EE 1.4 projects only)
    • CMP Entity Beans from Database (J2EE 1.4 projects only)
  4. Follow the directions in the rest of the wizard.

When you create an enterprise bean, the IDE does the following:

  • Java EE 5 projects:
    • Creates the bean class and the interfaces under the Source Packages node
    • Opens the bean class in the Source Editor
  • J2EE 1.4 projects:
    • Adds a node for the enterprise bean under the project's Enterprise Beans node. This node contains a node for the bean class and nodes for the bean's business methods and CMP fields.
    • Updates the deployment descriptors for the EJB module and the Java EE module that contains the EJB module. If your target server is the Sun Java System Application Server/GlassFish, the IDE also configures the server-specific deployment descriptors. For all other application servers, you have to write the server-specific deployment descriptors yourself.
    • Opens the bean class in the Source Editor
