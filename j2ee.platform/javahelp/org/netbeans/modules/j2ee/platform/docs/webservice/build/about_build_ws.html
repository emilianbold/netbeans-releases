About Building Web Services

In general, you build a web service implemented within a EJB module or a web application just as you would build any other project. As with other projects, the IDE automatically detects whether a project is part of a J2EE application and sets whether to package the archive files on the classpath accordingly. The IDE does not package API libraries like the Servlet API in your EJB module JAR file. You can adjust which archive files are packaged with your module in the module project's Project Properties dialog box.

However, the wscompile tool is run as part of the build process. On the service side, the wscompile tool generates a WSDL file from a web service or a web service from a WSDL file. The wscompile tool provides features that you can set to optimize the resulting WSDL file or web service. For example, you can set features to specify the style of your web service.

  For simple web services and web service clients, setting the wscompile tool's features is not required, because the default files generated by the IDE are generally sufficient. Only more complex web service interactions require you to set wscompile tool features.

To build the web service:

  1. Do one of the following to build the web service:
    • If the module within which the web service is implemented is part of a J2EE application, make sure the J2EE application is set as the main project and choose Build > Build Main Project (F11).
    • If the project is a stand-alone module, or if you want to build the module without building the other modules in the J2EE application, right-click the project's node in the Projects window and choose Build Project.
  2. Analyze the result and, if necessary, customize the wscompile tool's features, depending on your needs:
    • Building a Web Service from a WSDL File
    • Building a Document-Style Web Service
    • Building a RPC-Style Web Service
