Issue Tracking: Create Issue Tracker Dialog Box: JIRA

See Also

The Create Issue Tracker dialog box enables you to specify the connection details for an issue tracker. After you create a tracker, a node for the tracker appears under the Issue Tracker node in the Services window and you can view, submit and resolve issues in the IDE.

You see this dialog box when you right-click the Issue Trackers node in the Services window and choose Create Issue Tracker and select JIRA in the Connector dropdown list.

Note.

  • To enable integrated support for JIRA issue trackers in the IDE you need to install the JIRA plugin. You can click Download JIRA plugin in the Create Issue Tracker dialog box to install the JIRA plugin from the NetBeans Update Center. You will need to restart the IDE to complete installation of the plugin.

    If the JIRA plugin is not installed and you try to view or work with issues that are tracked using JIRA, the IDE will prompt you to use the Plugins manager to install the JIRA plugin.

  • If you do not want to install the plugin you can work directly with JIRA issues in your browser. You can find the location of the JIRA issue tracker in the details window of the project.

To register a JIRA issue tracker, specify the following properties.

  • Connector. Select JIRA from the dropdown list.
  • Name. The name of the tracker as it will be displayed in the IDE under the Issue Tracking node in the Services window. The name will also be listed in the drop down menu when you open the Find Issues tab from the Services window.
  • Issue Tracker URL. Specify the URL for the tracker.
  • Username. Specify the username that is registered with the issue tracker.
  • Password. Specify the username that is registered with the issue tracker.
  • Use HTTP Authentication. Select if the issue tracker requires authentication. If selected, you need to specify the username and password. Click Validate to connect to the tracker and check that the authentication details are correct.
  • Enable Local Usernames. When selected, the IDE uses the user name (e.g., without @mydomain.com) instead of an email address when submitting issues.

    When authorizing and identifying users, some issue trackers are configured to use a registered user name as the user id and some are configured to use an email address. Select this option when the issue tracker is configured to use a user name.

Perform the following steps to install the JIRA plugin using the Plugins Manager.

  1. Open the Plugins manager by choosing Tools > Plugins from the main menu.
  2. Select the Available Plugins tab.
  3. Locate the JIRA plugin from the list of available plugins.

    You can use the Search field in the Plugins manager to help you locate the plugin.

  4. Select the Install checkbox for the JIRA plugin.
  5. Click Install.

When you click Install, the IDE will open the installer. Proceed through the installation process to install the plugin and restart the IDE.

Note.

  • You will need to restart the IDE to finish installing the plugin.
