Issue Tracking: Create Issue Tracker Dialog Box

See Also

The Create Issue Tracker dialog box enables you to specify the connection details for an issue tracker. After you create a tracker, a node for the tracker appears under the Issue Tracker node in the Services window and you can view, submit and resolve issues in the IDE.

To create an issue tracker, specify the following properties.

  • Connector. Select the type of issue tracker from the drop down list.
  • Name. The name of the tracker as it will be displayed in the IDE under the Issue Tracking node in the Services window. The name will also be listed in the drop down menu when you open the Find Issues tab from the Services window.
  • Issue Tracker URL. Specify the URL for the tracker.
  • Username. Specify the username that is registered with the issue tracker.
  • Password. Specify the username that is registered with the issue tracker.
  • Use HTTP Authentication. Select if the issue tracker requires authentication. If selected, you need to specify the username and password. Click Validate to connect to the tracker and check that the authentication details are correct.
  • Enable Local Usernames. When selected, the issue tracker uses the short form of usernames (e.g., without @mydomain.com) when submitting issues.
  • Your username and password must be registered with the issue tracker to add the tracker.
