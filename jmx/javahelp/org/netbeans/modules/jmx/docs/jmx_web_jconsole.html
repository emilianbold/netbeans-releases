Bundled Tomcat configuration

In order to enable a JConsole access to the Tomcat JVM management data, the Tomcat launch script must be updated.
1) Edit <your netbeans install>/enterprise1/jakarta-tomcat-5.5.7/bin/catalina.bat (.sh on Unix)
2) Add -Dcom.sun.management.jmxremote  system property to Tomcat JVM run command line. For each exec command, add the needed property. Follows a catalina script extract :

...
 exec "$_RUNJAVA" $JAVA_OPTS $CATALINA_OPTS \
      -Djava.endorsed.dirs="$JAVA_ENDORSED_DIRS" -classpath "$CLASSPATH" \
      -Djava.security.manager \
      -Djava.security.policy=="$CATALINA_BASE"/conf/catalina.policy \
      -Dcatalina.base="$CATALINA_BASE" \
      -Dcatalina.home="$CATALINA_HOME" \
      -Djava.io.tmpdir="$CATALINA_TMPDIR" \
      -Dcom.sun.management.jmxremote \
      org.apache.catalina.startup.Bootstrap "$@" start
...

3) You are done. You are now able to visualize your MBeans (and all other JVM MBeans) from JConsole.

Connecting JConsole

1) First, be sure to have properly configured Tomcat.
2) If you have a running Tomcat, stop it.
3) Run your web application.
4) Call Run/Start JConsole... (or its associated Toolbar icon).
5) JConsole is displayed.
6) Select org.apache.catalina.startup.Bootstrap from the proposed table of Agent to connect to.
7) Click Connect.
8) You can now visualize your MBeans as well as Tomcat and JVM ones.
