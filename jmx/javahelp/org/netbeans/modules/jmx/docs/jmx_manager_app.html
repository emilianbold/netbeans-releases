Writing a JMX Manager


This chapter demonstrates the use of the JMX Manager Wizard. Part of the contained information can be applied to any Java application.

Any application that connect to a remote JMX Agent is a JMX Manager. This Wizard handles a connection to a remote JMX Agent.

A tutorial is accessible from http://management.netbeans.org/jmx
It details the needed steps to write and connect a manager application to a JMX Agent.

Using the JMX Manager Wizard to generate a  Manager

  1. Select File/New File...
  2. Select the Management category. In File Types select JMX Manager
  3. Type in a Name and a Package.
  4. A main method and some sample code are generated. If the current project is a Java Application project, the agent class will be set as the project main class.  You can disable these features by unselecting the appropriate checkbox.
  5. Click Next>
  6. In order to identify the agent you whish to connect to, click on Edit... button to enter a JMX agent URL
  7. A JMX URL is composed of a protocol, a host, a port and a URL path.
  8. First, enter a protocol. If you want to connect to a JDK 1.5 Agent, select "RMI JVM Agent". JMX module relies on this protocol. If you want to connect to agent generated with this JMX module you should select this protocol. You can enter any other protocol (eg: jmxmp)
  9. Then a host.
  10. A port.
  11. A URL path. In case of RMI JVM Agent the URL path is disabled and automaticaly computed by the wizard.
  12. Click OK when you are done. The JMX Agent URL textfield is updated with your inputs. You can click on Edit... again to update the URL.
  13. If the connection to the JMX Agent you selected requires authentication, you can provide some credentials. Select Generate Authenticated Connection with User Credentials checkbox
  14. Enter a username and password.
  15. If your connection is not authenticated  but you are interested in some sample code in order to provide such credentials, select Generate Sample Code for Authenticated Connection checkbox.
  16. Click Finish>
  17. The JMX Manager source code is loaded in the editor.
  18. If you want to get familiar with the generated code, go to the manager connect method. You will see how JMX API is used to connect to a remote Agent. Connection based on the URL and credentials you provided.
  19. If you selected Generate Sample MBean Discovery Code, go to the main method. Uncomment the generated code. This code make a call to the remote agent in order to discover the running MBeans. The list of discovered MBean is displayed on the output.
  20. You can then run your project like any other Java project (menu Run > Run Main Project). If a JMX Agent is up and running, you should see the list of discovered MBeans displayed in the output console.
