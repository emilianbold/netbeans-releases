IBM WebSphere Application Server Properties

See Also

You access this property sheet by right-clicking an IBM WebSphere application server instance node in the Services window and choosing Properties.

  • The Username and Password fields are the only properties you can modify. All the other properties are read-only.

Properties

  • Server Name. Shows the name of your application server for display purposes in the Services window. This property is set when you register your application server in the IDE using the Add Server Instance wizard.
  • Server Type. Shows the type of application server. This property is set when you register your application server in the IDE using the Add Server Instance wizard.
  • Connection tab. Shows the connection and configuration information about the server instance. This information is set when you register your application server in the IDE using the Add Server Instance wizard.
    • Local/Remote. Shows whether this is a local or remote server instance.
    • Local Instances. Shows the local server instances.
    • Profile Path. Shows the path to the root directory of the profile of the server instance.
    • Host. Shows the host name for the application server.
    • Port. Specifies the number of the TCP/IP port that the server listens on.
    • Username and Password. Specify the username and password the IDE will use when starting the server. The value you supplied when you registered the server instance is the default value for this field.
