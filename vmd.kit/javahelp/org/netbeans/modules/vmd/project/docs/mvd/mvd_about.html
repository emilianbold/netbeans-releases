About Visually Designing MIDP Applications

See Also

The Visual Mobile Designer (VMD) enables you to visually design your application's work flow and graphical user interface (GUI).

Tips for using the Visual Mobile Designer:

  • Begin your application by using VMD templates, available from the New File wizard.
  • Drag and drop objects into the Flow or Screen Designer windows to create your application.
  • Saving the file or switching to Source view prompts the IDE to generate and update the Java code of your design.
  • The VMD cannot edit GUI forms that were created outside of the IDE.
  • Using an external editor to modify the VMD generated code within the gray shaded areas may disable the ability to work on the project within the VMD.

The VMD provides the following tools to simplify the process of designing your MIDP applications, including:

  • Flow Designer. The primary workspace for the VMD. In this space, you map out the flow of the application by adding screens and other UI components, and associating commands with the components.
  • Screen Designer. Enables you to design and modify screens within your application.
  • Navigator Window. Displays a tree hierarchy of all components contained in the currently active application, including screens, commands, and resources.
  • Component Palette. Contains the Java objects you can add to your application.
  • Component Palette Manager. Enables you to add your own screens and objects to the Component Palette.
  • Properties window. Displays the editable settings for the currently selected component.
  • Analyzer. Checks your visual design for unused components and reports all components, properties, and values that are not available in MIDP 1.0.
