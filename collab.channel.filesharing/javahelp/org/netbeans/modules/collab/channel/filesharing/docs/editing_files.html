Editing Shared Files

See Also

Not only can you share either Project or non-Project files with other developers, but most files are also editable in real time. This means that you and your colleagues can edit a file together simultaneously. Similar to the way a database table prevents clients from overwriting the same records at the same time by locking rows, a change to a part of a shared file is protected by a guarded region so that only one person at a time may modify that part of the file.

Files you share can be files that are part of projects you have created., or are non-Project files, When you share a file that is not part of a project, the file is placed in a structure in your Shared Files Pane called "Shared Common Files".

Types of files that are not parts of projects and which would be shared under Shared Common Files include: jar files and zip files.

  Before a file can be edited, it must be shared. See Sharing Files for more information.

Keep in mind that although a file is sharable, it may not be editable. In general, if a file can be edited locally in the IDE, it will be editable when shared. The following are examples of editable and non-editable files:

To add a new file to the Shared Common Area:

  1. Right-click the root node and select New, then select Folder.
  2. Provide a name for the folder.
    The folder is created under Shared Non-project files.
  3. Right-click the folder to create an empty file, and provide a File Name for the new file.

To edit a shared file:

  1. Share (or have a colleague share) the file you want to edit within a conversation.
  2. Open the shared file by double-clicking the file or by using the contextual menu to invoke the Open action.
    The opened file has an annotation on its document tab to show that it is a shared file.
  3. Begin editing the file.
    As you make changes to the file, other participants will see a colored highlight around the region where you are making changes, and they will be unable to change the same section of the file. However, they may change other (unlocked) areas of the file while you are making your changes.
    Other participants will not see your changes until the area you are changing is unlocked. Changed regions of the file are automatically unlocked when you stop editing for a short time, or when you move out of that region and begin editing another region. You can set the guarded region timeout in the File Sharing channel options under Tools > Options, in the Collaboration > Channels category.
  4. When you have finished editing the file, you can save the file and then drag or copy it back to a local filesystem to preserve any changes made during the conversation.
