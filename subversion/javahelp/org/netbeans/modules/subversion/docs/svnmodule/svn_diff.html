Comparing File Revisions in Subversion

See Also

You can compare two file revisions side-by-side by performing a diff. The Diff command allows you to compare two different revisions through a graphical interface.

Comparing File Revisions Graphically

The IDE's Graphical Diff Viewer enables you to compare different versions of a file side by side using color coding to emphasize the differences between the files being compared.

The Previous () and Next () difference buttons in the toolbar enable you to navigate among differences found in the file. You can also refresh the status and update files from within the Diff Viewer using the Refresh Diff () and Update () buttons. When you are finished comparing the files, you can commit your changes using the Graphical Diff Viewer's Commit button ().

The Graphical Diff Viewer also enables you to select whether to view files containing changes from the local working copy, the repository, as well as both simultaneously. To change viewing criteria, click on the appropriate icon:

  • : local working copy
  • : repository
  • : both local and remote files

The Graphical Diff Viewer highlights changes in the files using the following colors:

  • Blue. Indicates lines that have been changed since the earlier revision.
  • Green. Indicates lines that have been added since the earlier revision.
  • Red. Indicates lines that have been removed since the earlier revision.

To generate a graphical diff comparing a revision to your working copy:

  1. Right-click the appropriate file node in the Projects, Files, Favorites or Subversion window.
  2. Choose Diff from the right-click menu. The IDE displays the results in the Graphical Diff Viewer in a new tab of the main window.
